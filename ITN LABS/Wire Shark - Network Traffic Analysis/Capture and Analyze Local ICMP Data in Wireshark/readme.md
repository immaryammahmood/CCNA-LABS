<h1> Local Network ICMP Traffic Analysis Using Wireshark</h1>

### Overview

This lab focuses on capturing and analyzing **ICMP traffic within a Local Area Network (LAN)** using Wireshark. The objective is to understand how devices communicate inside the same network by examining packet details such as IP addresses, MAC addresses, Ethernet frames, and ICMP messages.

ICMP traffic was generated using the `ping` command between local devices, and Wireshark was used to capture and analyze the packets.

---



### Lab Environment

| Category | Details |
|----------|---------|
| **Tools Used** | Wireshark, Windows Command Prompt |
| **Network Environment** | Local Area Network (LAN) |
| **Protocols Analyzed** | ICMP, IPv4, Ethernet II, ARP |

Examine the data that was generated by the ping requests of your team member PC. Wireshark data is displayed in three sections: 

1) The top section displays the list of PDU frames captured with a summary of the IP packet information listed.
