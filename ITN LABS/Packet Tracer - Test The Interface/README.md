# Test The Interface - Packet Tracer Activity

## Overview

This Packet Tracer lab file is designed to help you practice verifying network device configurations and connectivity using CLI commands. After implementing basic IP addressing on switches and PCs in previous labs, this activity focuses on **validating that those configurations are actually working** through verification commands.

## What's Covered

This simulation environment demonstrates how to verify interface status and IP address configurations on networking devices using industry-standard commands. The key concept here is that configuration alone doesn't guarantee functionality—verification is essential to ensure your setup is correct.

### Key Commands Demonstrated

- **`show ip interface brief`** - Used on switches and routers to quickly view the status of all interfaces, their IP addresses, and whether they're up or down
- **`ipconfig`** - Used on PCs to verify their assigned IP addresses and network configuration

## Why This Matters

In real-world network administration, verification is a critical skill. After you configure a device, you need to confirm:
- Interfaces are operational (up/up status)
- IP addresses are correctly assigned
- Connectivity between devices is working as intended

This lab provides a hands-on opportunity to practice these verification techniques in a safe, simulated environment using Cisco Packet Tracer.
